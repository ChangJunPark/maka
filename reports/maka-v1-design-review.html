Design Review Gate · Maka v1 candidate

기능은 v1 후보, 화면은 한 번 더 정돈 필요

현재 Maka는 로컬 폴더 기반 Markdown 작업 흐름을 갖췄다. 디자인 리뷰의 기준은 “예쁜 앱”보다 “문서 작성 중 사용자가 상태를 놓치지 않고, 파일 작업이 안전하게 보이며, 터미널/미리보기가 방해되지 않는가”다.

현재 정보 구조

Topbar: workspace, layout toggle, folder open, save
Files: list + create/rename/delete
Editor: toolbar + CodeMirror
Preview: Markdown + lazy Mermaid
Terminal: manual workspace shell
Statusbar: message + shortcut hints

리뷰 결과

좋음

작업 영역 구분

파일/편집/미리보기/터미널이 명확히 분리되어 VS Code식 작업 모델을 빠르게 이해할 수 있다.

좋음

상태 안전성

충돌 banner, status pill, save disabled 상태가 있어 자동저장/외부변경의 위험을 숨기지 않는다.

보완

Toolbar 밀도

파일 버튼과 editor toolbar가 pane title 안에 있어 작고 빽빽하다. v1 이후 icon/tooltip 정리가 필요하다.

보완

터미널 높이

하단 터미널은 유용하지만 고정 높이라 글쓰기 집중 모드에서는 다소 크게 느껴질 수 있다.

보완

상태 메시지 지속성

하단 message는 좋지만 최근 저장 시각이나 외부 변경 시간처럼 더 구체적인 timestamp가 있으면 좋다.

주의

실제 화면 검증 필요

이 리뷰는 코드/구조 기반이다. 최종 전 실제 macOS 창 screenshot으로 spacing과 버튼 가독성을 한 번 더 봐야 한다.

v1 전 디자인 체크리스트

  • 파일명이 긴 경우 file pane에서 잘리는 방식 확인
  • 충돌 banner가 toolbar를 가리지 않는지 확인
  • 편집-only / preview-only toggle이 즉시 이해되는지 확인
  • 새 파일/rename/delete 버튼이 너무 위험해 보이지 않는지 확인
  • terminal disabled/idle/running 상태가 구분되는지 확인
  • Mermaid diagram loading/error 상태가 문서 흐름을 깨지 않는지 확인
  • statusbar shortcut hint가 좁은 창에서 과밀하지 않은지 확인
  • 실제 app bundle에서 font/spacing이 dev와 같은지 확인

결론

v1 후보로는 충분하다. 다만 “출시 polish” 단계에서는 toolbar를 icon 기반으로 바꾸고, 터미널 높이 조절, 저장 timestamp, 실제 screenshot 기반 visual pass를 추가하는 것이 좋다.
